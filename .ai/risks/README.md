# Risk Register

This directory stores identified risks for the project.

## When to Log a Risk

Log a risk here when:
- A dependency (external service, team, approval) could delay a phase
- A technical unknown could require rework
- A QA or security finding is accepted rather than resolved
- A scope decision introduces technical debt with known future impact

## Risk Entry Format

```markdown
## RISK-001 — Short Title

**Identified:** YYYY-MM-DD
**Phase:** (phase where risk was identified)
**Severity:** Critical / High / Medium / Low
**Probability:** High / Medium / Low
**Status:** Open / Mitigated / Accepted / Closed

### Description
What the risk is.

### Impact
What happens if it materializes.

### Mitigation
What action reduces this risk.

### Owner
Which agent or role monitors this risk.
```
