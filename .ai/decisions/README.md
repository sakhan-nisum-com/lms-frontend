# Decision Log

This directory stores key architectural and product decisions made during the project.

Use the ADR template at `templates/adr.md` for each decision.

## When to Log a Decision

Log a decision here when:
- A design choice has meaningful alternatives that were considered and rejected
- A standard is deviated from with intentional justification
- A technical approach is chosen that future developers might question
- A scope or product decision is made that affects what gets built

## Naming Convention

```
YYYY-MM-DD-short-slug.md

Example:
2024-03-15-uuid-primary-keys.md
2024-03-20-soft-delete-pattern.md
```
